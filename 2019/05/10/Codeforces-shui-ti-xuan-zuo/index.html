真·水题

【CF319E】Ping-Pong

【CF319E】Ping-Pong
考虑区间 $ [a, b] $ 和区间 $ [c, d] $ 之间连的是单向边还是双向边。
若 $ a < c < b < d $ 或 $ c < a < d < b $ ，则连的是双向边，即可以互相到达，用并查集维护，线段树维护区间关系，由于一个点只会被插入一次、删除一次，所以复杂度正确。
若两个区间为包含关系，则为单向边，这时候不好判断。考虑一个区间所在并查集的区间并是一个大区间，所以询问时只需要比较两个区间所在的并查集大区间是否有交即可。

【CF700D】Huffman Coding on Segment

【CF700D】Huffman Coding on Segment
单纯的暴力就是统计一段区间出现的每种数的个数，然后放在一个堆里贪心。
统计一段区间出现的每种数的个数显然可以用莫队维护，但计算答案十分麻烦。
考虑出现次数大于 $ \sqrt{n} $ 的数的个数不超过 $ \sqrt{n} $ 个，所以将出现个数不大于 $ \sqrt{n} $ 开个桶暴力扫描（计算过程中大于 $ \sqrt{n} $ 后丢堆里），否则放在一个堆里贪心。
时间复杂度 $ O(n \sqrt{n} \log n) $ 。

【CF235D】Graph Game

【CF235D】Graph Game
设 $ f[i][j] $ 为 $ i $ 被选为分治点时与 $ j $ 连通的概率，则 $ Ans = \sum {f[i][j]} $ 。
对于一棵树， $ f[i][j] = \frac{1}{dist(i, j)} $ ，即 $ i $ ，即 $ i $ 为 $ i $ 到 $ j $ 第一个选择的点。
然而这是一颗基环树。对于不经过环的点对，则 $ f[i][j] = \frac{1}{dist(i, j)} $ ，适用。否则设将环分成长度为 $ x, y $ 的两部分，通过容斥可以得出 $ f[i][j] = \frac{1}{dist(i, j) + x} + \frac{1}{dist(i, j) + y} - \frac{1}{dist(i, j) + x + y} $ 。
暴力枚举根，然后暴力 $ dfs $ ，复杂度 $ O(n^2) $。

【CF1088F】Ehab and a weird weight formula

【CF1088F】Ehab and a weird weight formula
将点的代价转为边的代价，则边的代价为 $ \lceil\log2{dist(x, y)}\rceil \times \min(v_x, v_y) + v_x + v_y $ 。
考虑将点权最小的点作为根，存在结论 $ v_u > v{fa[u]} (u \neq root) $ 。证明即若存在一个儿子节点权值比当前节点小，则这个儿子节点的某个儿子权值比这个儿子节点还小，最后的叶子节点会很尴尬。
如此边权计算中的 $ \min $ 也可去掉。
那显然改造过的树每个节点的新父亲要么是根，要么是第 $ 2^k $ 个祖先。暴力 $ dfs $ 并找祖先，时间复杂度 $ O(n \log n) $ 。
Code

【CF1110G】Tree-Tac-Toe

【CF1110G】Tree-Tac-Toe
黑方根本不可能赢。
如果有一个点度大于3或度为3且有其中两个儿子度大于1则白方赢。
如果有白点，相当于存在一个度为3的点连向这个点（没有颜色）。
特判一种像骨头然后中间有奇数个点的情况，这种情况白方赢。
否则平局。

【CF653G】Move by Prime

【CF653G】Move by Prime
每个质数是互相独立的。处理出每个数包含某个质数的多少次，进行若干次操作后质数的指数选中位数最优。
有 $ n $ 个物品，两两不相同。它们被分成两堆，左边那堆 $ x $ 个，右边那堆 $ n−x $ 个。从左边那堆拿出 $ p $ 个物品，从右边拿出 $ q $ 个物品，要求 $ p > q $ 。
方案数为 $ \sum_{i = 0}^{x - 1} \binom{n}{i} $ ，证明链接。
维护组合数前缀和即可。
Code

【CF1030G】Linear Congruential Generator

【CF1030G】Linear Congruential Generator
每个函数进行 $ 0 $ 或 $ 1 $ 次变换会进入一个循环，形如 $ \rho $ 状。
设第 $ i $ 个函数环长 $ c_i $ ，链长 $ p_i $ ，最后答案为 $ lcm{c_1, c_2, \dots , c_n} + max{p_1, p_2, \dots , c_n} $ 。

【CF453E】Little Pony and Lord Tirek

【CF453E】Little Pony and Lord Tirek
特殊考虑还未被吸收过的马。
对于被同一时间吸收的马，将马按恢复完成所需时间从小到大排序。显然恢复完成所需时间不大于经过时间的马的魔力已经满了，否则等于效率乘时间。维护前缀魔力Max和、后缀效率和，询问时二分即可。
可以用 $ ODT $ 或线段树维护。

【CF461E】Appleman and a Game

【CF461E】Appleman and a Game
考虑如何构造 s 串使得 Appleman 最难受。假设当前 $ s[1 \to u] $ 串已经被匹配，则选择最短的 $ s[u + 1 \to len] $ 串，使得 $ s[u + 1 \to len] $ 在 t 串中出现且 $ s[u + 1 \to len + 1] $ 不在 t 串中出现。
由于随着操作次数增加， s 串变长，所以考虑二分答案，即判断若干次操作后 s 串长度是否可能不大于 $ n $ 。
由于 $ n $ 很大，所以不能用递推维护，可以选择矩阵乘法。
Code

【CF504E】Misha and LCP on Tree

【CF504E】Misha and LCP on Tree
字符串比较选用好写常数小的 $ hash $ 。用 $ hash $ 维护一个点往上跑到根与从根跑到这个点的 $ hash $ 值，询问一条自上向下的链时可以用差分维护。
将树进行树链剖分，路径被分成数量为 $ \log n $ 级别的向上或向下的段，然后逐段比较。
由于需要考虑到行走方向，所以代码量大。
Code

【CF587F】Duff is Mad

【CF587F】Duff is Mad
建出AC自动机的fail树，一个节点的祖先是这个节点表示的串的后缀。
存在两种暴力：

  1. 将 $ sk $ 每个位置打上标记，然后对于 $ s{1 \dots n} $ 询问 $ endpos $ 的子树中有多少个标记。
  2. 先将询问作前缀和差分处理，将 $ s_{1 \dots n} $ 逐个将 $ endpos $ 对应子树加上区间标记，对于每一个询问串，计算每一个前缀对应在fail树上的位置上标记个数之和。

由于长度不小于 $ \sqrt{n} $ 的串至多有 $ \sqrt{n} $ 个，所以对于长度不小于 $ \sqrt{n} $ 的串用第一种暴力，否则用第二种暴力。
由于询问次数远大于修改次数，所以用分块维护前缀和即可。
Code

【CF274E】Mirror Room

【CF274E】Mirror Room
发现障碍物是在 $ n $ 的数量级的，容易发现最后经过的路径要么是一条不相交的回路，要么是一条不相交的线，碰壁次数不多（大概数 $ 4 \times n $ 次）。
每次二分找到反弹点，大力分类讨论即可。

【CF626G】Raffles

【CF626G】Raffles
考虑对于一种彩票，设已经投注 $ num_i $ 次，多投注一次对答案的贡献是 $ S = p_i \times (\frac{num_i + 1}{l_i + num_i + 1} - \frac{num_i}{num_i + l_i}) $ ，发现随着 $ num_i $ 的增大 $ S $ 在减小。
考虑每次贪心每次取贡献最大且可以合法投注的奖池投注，最后答案显然最优。
每次修改只会对 $ l_i $ 进行 $ +1/-1 $ 的操作，对投注方案的修改极少。当手上还有票时直接选择投注，否则与当前已投注贡献最小的比较并判断是否取代。
修改 $ num_i $ 、查找贡献最大/最小可以用线段树维护。
Code

【CF771E】Bear and Rectangle Strips

【CF771E】Bear and Rectangle Strips
由于只有两列，我们可以用map找到第一列、第二列以及两列合在一起的矩形合法的左端点最靠右的位置。
设 $ dp[i] $ 表示前 $ i $ 列最多有几个矩形，通过记忆化搜索完成DP即可通过此题。

【CF1063F】String Journey

【CF1063F】String Journey
这里有各种做法
有一种 $ O(n \sqrt{n}) $ 的做法。考虑存在一种方案， $ |t1| = |t_2| + 1 $ ，即 $ t_1 $ 去掉开头或去掉结尾一个字符得到 $ t_2 $ 。
枚举长度从后向前枚举串，用哈希值比较后面是否存在一个符合条件的 $ t{i + 1} $ （ $ fi = f{i + 1} + 1 $ ， $ lcp(ti, t{i + 1}) = |t{i + 1}| $ 或 $ lcs(t_i, t{i + 1}) = |t_{i + 1}| $ ）。

【CF1103D】Professional layer

【CF1103D】Professional layer
考虑真正有用的素因数是所有数的 $ \gcd $ 所包含的素因数。显然这样的素因数不会超过 $ 12 $ 个。
考虑每多选出一个数，素因数个数会至少减一，所以做多选 $ 12 $ 个数。
考虑一个数包含 $ \gcd $ 的多少（设为 $ d $ ），对于同样一个 $ d $ ，显然会贪心地取前 $ 12 $ 个。
状压DP维护选了几个数、还有那些素数没被去掉的最小代价。
