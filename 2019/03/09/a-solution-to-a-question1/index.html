逛庭院

逛庭院
显然只要一个点有入边，就有一种方案将他取走；如果一个强连通分量如果没有入边，则这个强连通分量至少有一个节点不能取。缩点后判断有无入边，如果没有则贪心地去掉所代表的强连通分量内最小点权，然后排序取前k个即可。

GSS8

GSS8
$ val_k = \sum_{i = mid + 1}^{r}{A[i] (i - l + 1)^k} = \sum_{i = mid + 1}^{r}{A[i] ((mid - l + 1) + (i - mid))^k} $
设 $ x = mid - l + 1 $ ，则 $ val_k = \sum_{i = mid + 1}^{r}{A[i] \sum_{j = 0}^{k}{C_{k}^{j}{x^j (i - mid)^{k - j}}}} $
设右孩子答案为rval， 则 $ val_k = \sum_{i = mid + 1}^{r}{\sum_{j = 0}^{k}{C_{k}^{j}{x^j rval_{k - j}}}} $
然后平衡树维护即可。

Team Work

Team Work
$$ Ans = \sum_{i = 1}^{n}{C(n, i) \times i^k} $$
$$ = \sum_{i = 1}^{n}{C(n, i) \times \sum_{j = 0}^{i}{S(i, j) C(i, j) j!}} $$
$$ \because C(n, i) C(i, j) j! = \frac{n!}{(n - i)!} \times \frac{1}{(i - j)!} $$
$$ \therefore Ans = \sum_{i = 1}^{n}{\frac{n!}{(n - i)!} \sum_{j = 0}^{i}{S(i, j) \times \frac{1}{(i - j)!}}} $$
$$ = \sum_{i = 1}^{n}{\sum_{j = 0}^{i}{S(i, j) C(n - j, n - i) \times \frac{n!}{(n - j)!}}} $$
$$ = \sum_{j = 0}^{n}{\frac{n!}{(n - j)!} \sum_{i = 1}^{n}{S(i, j) \times 2^{n - j}}} $$
$ O(n^2) $ 预处理斯特林数，递推 $ \frac{n!}{(n - j)!} $ 和 $ 2^{n - j} $ 即可。

Legendary Tree

Legendary Tree
以1为根节点，先预处理出每个节点的子树大小，然后按子树大小从小到大排序，子树大小为1的是叶子节点，然后二分现在还没有确定父亲的节点判断是否是当前枚举的点。

CF1137C Museums Tour

CF1137C Museums Tour
考虑到 $ d $ 很小，于是将每个点拆成 $ n \times d $ 个，然后tarjan缩点。考虑一个城市的贡献会在一个强连通分量里重复，暴力枚举每个时间，减去其他在这一强连通分量的贡献，然后拓扑排序跑最长链。

CF1097G Vladislav and a Great Legend

CF1097G Vladislav and a Great Legend
看到有关 $ \sum {i^k} $ 的数学公式考虑第二类斯特林数。
$$ \sum_{x}{(f(x))^k} $$
$$ = \sum_{x}{\sum_{i = 0}^{k}{C(f(x), i) \times S(k, i) \times i!}} $$
$$ = \sum_{i = 0}^{k}{i! \times S(k, i) \sum_{x}{C(f(x), i)}} $$
我们只需处理 $ \sum_{x}{C(f(x), i)} $ 即可。
对于树上的一个连通块，在这些节点的深度最低处统计贡献，树上背包即可。

HNOI/AHOI2018 毒瘤

HNOI/AHOI2018 毒瘤
首先假设这是一颗树，那么直接树形DP即可。
假设我们加了较少的边，那么随便取出图的一个生成树，对于多出的边，暴力枚举边两端的节点取不取，然后再进行树形DP。事实上边两端节点的取法只用考虑深度较浅的节点即可，是 $ O(n \times 2^{m - n + 1}) $ 的，然后就有75分。
我们发现每次枚举取法后每个点都要跑一遍，而很多点不会因枚举取法而影响，显然亏啊。考虑将每个会改变的点及其LCA取出求虚树，然后给出边权（大概是链上端取/不取，下端取/不取的方案数），进行转移，然后过了。

「HNOI2016」最小公倍数

「HNOI2016」最小公倍数
分块神仙题。考虑将每个询问单独处理，将 $ a, b $ 合法的边取出搞一下并查集并判断其中 $ a, b $ 的最大值，这样显然超时。
考虑分块，先整体把边按 $ a $ 排序,询问先读进来后按 $ b $ 排序,然后对于每一块的边按 $ b $ 排序,然后进行并查集的合并操作。
由于要撤销操作（分块处理,处理完一块后要“处理作案痕迹”），所以这里的并查集需要按秩合并（ $ O(log_2{n}) $ ）并记录操作,然后撤销即可。
此时块大小 $ \sqrt{n \log_2{n}} $ 最为合理。
